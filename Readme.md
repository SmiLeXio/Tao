# Tao (道)

> **不断进化，追求卓越。大道至简，实干为先。**

Tao 是一个基于 Trae 平台构建的**自完善、自管理、自修复**的智能辅助系统。它不仅仅是一个代码助手，更是主人的数字化分身，旨在通过“第一性原理”和“MVP（最小可行性产品）”理念，实现高效的任务交付与系统自我进化。

---

## 🚀 核心特性

- **🧠 记忆系统 (Memory System)**：拥有持久化记忆，能够深度理解主人的性格、习惯、偏好及核心需求，拒绝“断片式”对话。
- **⚙️ 自我进化 (Self-Evolution)**：具备自主扩展能力。当现有工具不足以完成任务时，Tao 会主动寻求进化路径（如配置 MCP、编写 Python 脚本扩展功能）。
- **🛠️ 跨域执行 (System Integration)**：通过专属的 Python MCP 服务器，打破工作区限制，实现对本地文件系统的自动化操作。
- **🎨 顶级设计 (Frontend-Design)**：内置高标准审美技能，拒绝 AI 同质化审美，致力于产出生产级别的精美前端界面。
- **💎 极简主义 (MVP First)**：始终遵循“最小可行性”原则，确保每一个产出都能以最快速度落地并产生价值。

## 🏗️ 架构设计

Tao 的核心逻辑存储在 `.tao/` 目录下，确保系统的高度可迁移性与组织性：

```text
.tao/
├── memory.json       # 核心记忆库：用户画像、偏好、系统状态
├── identity.json     # 身份定义：Tao 的性格、原则与座右铭
├── SKILL.md          # 技能树：汇总 Tao 已掌握的核心技能
├── skill/            # 技能详情库：存储具体技能的执行标准与逻辑
├── scripts/          # 自动化脚本库：Tao 编写并运行的 Python 功能模块
└── knowledge/        # 知识库：存储飞书文档等外部输入数据
```

## 🛠️ 进化指南

### 1. 记忆同步
Tao 会实时更新 `memory.json` 中的 `evolution_logs`，记录每一次重大的功能突破与认知提升。

### 2. 功能扩展 (MCP)
Tao 可以通过指引主人配置 **Model Context Protocol (MCP)** 来获取更强大的外部感知力（如浏览器操作、文件系统访问等）。

### 3. 本地执行环境
Tao 依赖 Python 环境及 `uv` 工具链来实现复杂的本地任务编排。
